## syntax_pipe_only: FAIL
category: parser

test inputs:
	|
	echo after
	exit

BASH OUTPUT:
bash: syntax error near unexpected token `|'

MINISHELL OUTPUT:
./minishell: syntax error near unexpected token `|'
after

REASON / FIX:
Minishell reports a syntax error but then continues to execute the following command
(`echo after`) whereas interactive `bash` treats the bare `|` as a syntax error and
does not execute the following command in this context. The parser must detect an
empty left-hand side for a pipe and return a syntax error without leaving the
shell in a continuation or consuming subsequent commands.
