Best Valgrind command for minishell
===================================

Use this as the strict evaluator command:

valgrind \
  --leak-check=full \
  --show-leak-kinds=all \
  --track-fds=yes \
  --trace-children=yes \
  --track-origins=yes \
  --errors-for-leak-kinds=definite,indirect \
  --suppressions=readline.supp \
  ./minishell


What must fail
==============

Fail the test if Valgrind reports any of these:

- definitely lost memory
- indirectly lost memory
- invalid read
- invalid write
- use of uninitialized value
- bad free / double free
- open file descriptors that should have been closed
- child-process leak not explained by readline or system libraries

Readline leaks are acceptable only when suppressed by readline.supp.
Do not excuse leaks from your own parser, env, expansion, heredoc, redirection,
pipeline, builtin, or executor code.


Recommended test style
======================

For one command:

printf 'echo hi\nexit\n' | valgrind \
  --leak-check=full \
  --show-leak-kinds=all \
  --track-fds=yes \
  --trace-children=yes \
  --track-origins=yes \
  --errors-for-leak-kinds=definite,indirect \
  --suppressions=readline.supp \
  ./minishell

For a longer script:

cat <<'EOF' | valgrind \
  --leak-check=full \
  --show-leak-kinds=all \
  --track-fds=yes \
  --trace-children=yes \
  --track-origins=yes \
  --errors-for-leak-kinds=definite,indirect \
  --suppressions=readline.supp \
  ./minishell
echo hello
export A="one two"
echo "$A"
echo hi | cat | wc -c
cat <<END
$A
END
exit
EOF


Minishell-specific Valgrind tips
================================

1. Always include --trace-children=yes.
   Without it, leaks and invalid memory in forked pipeline/external-command
   children can be missed.

2. Always include --track-fds=yes.
   Minishell commonly leaks pipe ends, heredoc fds, redirection fds, or saved
   stdin/stdout duplicates.

3. Test builtins both alone and in pipelines.
   Example:
   export A=1
   export A=2 | cat
   cd / | cat
   echo hi | cat

4. Test redirection failures.
   These often leak saved fds or partially allocated command structures:
   cat < /no/such/file
   echo hi >
   echo hi > /tmp/a > /tmp/b
   > /tmp/only_redir

5. Test heredoc cases.
   Heredoc code often leaks pipes or strings:
   cat << EOF
   hello
   EOF

   Also test quoted delimiters:
   cat << 'EOF'
   $HOME
   EOF

6. Test syntax errors.
   Parser error paths must free tokens, argv arrays, redirection lists, and
   partially built command lists:
   |
   echo >
   echo <
   echo hi > > file

7. Test exit paths.
   exit should free shell env and command memory:
   exit
   exit 42
   exit abc
   exit 1 2

8. Check the fd summary at the end.
   A clean shell should not leave project-owned fds open. Standard fds
   0, 1, and 2 are normal. Extra pipe/heredoc/redirection fds are failures.

9. Do not trust "still reachable" blindly.
   Readline may leave reachable allocations. Your own long-lived env may also
   be reachable until process exit if not freed. For 42 evaluation, free your
   own shell state cleanly and suppress only readline/system noise.

10. Run repeated loops.
    Some leaks only appear after many iterations:
    echo hi
    echo hi
    echo hi
    export A=1
    unset A
    echo done
    exit
